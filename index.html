Git

Comandos

git init: Iniciar um repositorio git
git ls-files: Visualizar arquivos do git
git status: Status do diretorio
git add: Adicionar o arquivo ao repositorio para rastrea-lo
git commit -m "Criando o projeto": Commitar com mensagem
git config user.name "Joao": Configura o nome do projeto git
git config user.email "joao@gmail.com": Configura o email do projeto git
git log: Mostra os commits e suas mensagens
git whatchanged: Mostra quais arquivos foi alterado
git whatchanged -p: Mostra o que foi alterado
git remote: Verifica se tem algum repositorio remoto
git remote add origin https://github.com/Lhuckaz/curso-git.git: Adiciona uma 'origin' no repositorio
git push origin master: realiza o commit no master

Estados do repositorio

Os três estados de um repositório são "Working Directory", que são os arquivos que temos em nosso repositório e podemos modificá-los, "Index" ou "Staging Area", que é uma visão preliminar das modificações em nossos arquivos que queremos definir como um novo passo do projeto, e "HEAD", que é um passo completo de nosso trabalho, usado como referência para a comparação com nosso "Working Directory".

Obs

Utilizando o comando git add

É possível passar mais de um arquivo como parâmetro para o comando git add, basta separá-los por um espaço em branco, no mesmo comando. Além disso é possível adicionar todos os arquivos da pasta atual, com git add ..

O parâmetro também pode ser uma operação de busca de arquivos comum, por exemplo, para adiconar todos os arquivos com a extensão "txt" ao Index, podemo utilizar o comando git add *.txt, ou todos os arquivos em uma determinada subpasta do repositório com git add nome_da_pasta/.

O comando git add interativo

O comando git add -i é mais recomendado quando temos um número muito grande de alterações a serem adicionadas ao Index, e também um grande número de alterações que não gostaríamos de adicionar ao Index. O modo interativo inclusive permite que adicionemos seletivamente trechos de um arquivo, deixando algumas modificações de fora mesmo estando em no mesmo arquivo.

A nomenclatura das opções não é consistente com os comandos que utilizamos até agora, porém perceba que sempre uma das opções é de ajuda, e podemos consultá-la a qualquer momento. Caso esteja em dúvida, sempre existe a opção de terminar o modo interativo e voltar ao prompt de comando

Commit automático

O comando git commit -a já cria um novo HEAD com todas as alterações detectadas no repositório, porém novos arquivos não serão adicionados. O comando pode ser utilizado diretamente, dessa maneira o editor de textos padrão do computador será aberto para que uma mensagem de "commit" seja adicionada.

Podemos combinar a opção de mensagem com git commit -am "Mensagem de commit".
